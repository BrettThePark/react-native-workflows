Module SecurityContextProvider

SecurityContextProvider provides a single source of state for the state of user authentication. It is not meant to authenticate the user or hold credential information. Its purpose is to control access to authenticated or non-authenticated sections of the application (as well as change password for a currently authenticated user).

SecurityContextState

  • Defined in src/contexts/SecurityContextProvider/types.ts:9

Basic state upon which to make application security decisions.

Type declaration

  • Optional email?: undefined | string

    Email of the authenticated user.

  • isAuthenticatedUser: boolean

    True: The user is authenticated and the application is shown (or the Change Password interface). False: The user is not authenticated and the Authentication User Interface is shown.

  • isLoading: boolean

    True: The security state is being loaded (all other fields are invalid). False: The security state has been loaded.

  • isShowingChangePassword: boolean

    True: The Change Password screen is currently visible. False: The Change Password screen is not currently visible.

  • isSignOut: boolean

    Used for animation purposes only. True: The user is logged in currently and a change will be the result of logging out. False: The user is likely logging in if authentication state changes.

  • rememberMeDetails: { email?: undefined | string; rememberMe?: undefined | false | true }

    Information for a user who wants to be remembered upon logout.

    • Optional email?: undefined | string

      Email address to show in the email field of Login after logout.

    • Optional rememberMe?: undefined | false | true

      When true, the user's email will be in the email field of Login.

  • Optional userId?: undefined | string

    UserId of the authenticated user (may be an email).
